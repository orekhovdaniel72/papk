# yuzotova-art — продуктовая спецификация

> Сервис для публикации резюме и портфолио 3D-художника.
> Версия документа: v1 (Этап 0). Обновляется по ходу проекта.

## 1. Видение

Личный сервис, где художница ведёт единую медиатеку работ (фото + видео) и
собирает из неё **несколько разных резюме/портфолио** под конкретные вакансии.
Каждое резюме шарится отдельной ссылкой так, чтобы получатель **не мог попасть
в соседние резюме** и **не мог скачать** медиа (реалистичная защита, см. §5).

## 2. Роли

- **Владелец (художница)** — единственный администратор. Логинится, управляет
  медиатекой, собирает резюме, создаёт ссылки для шаринга.
- **Получатель (работодатель)** — открывает конкретную ссылку (возможно, с
  паролем / сроком жизни). Видит только одно резюме. Без аккаунта.

## 3. Доменная модель (черновик)

- **MediaAsset** — единица медиатеки: фото или видео + метаданные (название,
  описание, теги, год, роль, софт). Хранится приватно (см. §5).
- **Resume (Portfolio)** — именованная подборка: заголовок, «обо мне», контакты,
  упорядоченный список работ (могут отличаться составом и сортировкой между
  резюме). Ссылается на MediaAsset (переиспользование без дублирования файлов).
- **ResumeItem** — связь Resume↔MediaAsset с порядком и опциональными
  переопределениями (подпись, обрезка обложки).
- **ShareLink** — токен доступа к ОДНОМУ Resume: уникальный slug, опц. пароль,
  опц. срок действия / лимит просмотров, отзываемость. Изоляция: токен открывает
  строго свой Resume, соседние недоступны.

## 4. Ключевые сценарии

1. Художница загружает работы в медиатеку.
2. Создаёт резюме «Под вакансию X», выбирает работы, задаёт порядок и тексты.
3. Генерирует ShareLink (опц. пароль + срок). Копирует ссылку, отправляет.
4. Работодатель открывает ссылку → видит красивую галерею, медиа стримится,
   скачать/сохранить штатно нельзя, поверх — водяной знак с ID ссылки.

## 5. Защита медиа — реалистичный подход (ВАЖНО, честно)

Полностью запретить скриншот/съёмку экрана на web **технически невозможно**.
Цель v1 — **максимально затруднить выгрузку + сделать утечки трассируемыми**:

- Приватное хранилище (Cloudflare R2), **никаких публичных URL**.
- Доступ к медиа только через **короткоживущие подписанные ссылки**, выдаваемые
  сервером после проверки токена ShareLink.
- **Видео**: стриминг (range / HLS по возможности), без отдачи цельного файла,
  `controlsList=nodownload`, отключение контекстного меню.
- **Фото**: рендер с защитой от drag/правого клика, без доступа к оригиналу в
  полном разрешении из разметки.
- **Водяной знак**: видимый (имя/ID ссылки полупрозрачно) + по возможности
  невидимый — чтобы по утёкшему скрину было видно, какой ссылке он принадлежал.
- Базовые анти-удобства для копирования (отключение выделения, drag, контекстного
  меню). Понимаем, что это деттеррент, а не абсолют.

Апгрейд-путь: при росте — заменить видео-слой на Cloudflare Stream / Mux (DRM,
сигнированный HLS) без переписывания остального.

## 6. Технологический стек (всё на бесплатных тарифах)

| Слой | Выбор |
|---|---|
| Фреймворк | **Next.js 16** (App Router, TypeScript) |
| UI | **Tailwind v4 + shadcn/ui** (radix, пресет nova: Lucide + Geist) |
| БД + Auth | **Supabase** (Postgres, free tier) |
| Хранилище медиа | **Cloudflare R2** (10 ГБ, нулевой egress) |
| Хостинг | **Vercel** (free) |

⚠️ **Next.js 16 новее тренировочных данных** — перед написанием кода сверяться с
`node_modules/next/dist/docs/`. См. AGENTS.md.

## 7. Не-цели v1 (осознанно откладываем)

- Платные/DRM видео-стриминг сервисы.
- Мультипользовательность (несколько художников).
- Аналитика просмотров (можно добавить позже как счётчик у ShareLink).
- Кастомные домены под каждое резюме.
